# Contributing to AhabOWAPI

Thank you for considering contributing to AhabOWAPI! 🎉

This project was created by a fellow learner, and it's meant to be **approachable for beginners and CompSci students**. Whether you're fixing a typo, improving comments, or adding a cool new feature — every contribution is welcome!

## How Can You Contribute?

- **Good First Issues** (great for beginners):
  - Improve or translate the README
  - Add more comments to the code
  - Fix typos or grammar
  - Suggest better variable/function names
  - Add input validation for the city field
  - Improve error messages

- **Other ways to help**:
  - Report bugs
  - Suggest new features
  - Add support for more weather data (humidity icons, wind direction arrows, etc.)
  - Improve the GUI layout
  - Add dark mode
  - Write tests

## Contribution Workflow

1. **Fork** this repository (click the Fork button at the top)
2. **Clone** your fork to your computer:
   ```bash
   git clone https://github.com/Sgt-Ahab/AhabOWAPI.git
   ```
3. Create a new branch for your changes:
```bash
git checkout -b your-awesome-feature
```
4. Make your changes and test them
5. Commit your changes:
```bash
git commit -m "Add better error handling for invalid ISO codes"
```
6. Push to your fork:
```bash
git push origin your-awesome-feature
```
7. Go to your fork on GitHub and click `Compare & pull request`

We'll review your Pull Request (PR) together. Don't worry if it's not perfect, we can work on it!

---

## Code Style & Guidelines
- Keep the code clean and well-commented (this is a learning project)
- Follow the existing style in `ahab_weather.py`
- Use meaningful variable names
- Test your changes before submitting


## Questions
Feel free to open an Issue if you have questions or need help getting started.
